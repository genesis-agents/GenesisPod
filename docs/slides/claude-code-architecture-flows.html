1. 总体架构分层 CORE

Claude Code 采用 6 层架构：入口层 → 状态层 → UI 层 → 查询引擎层 → 服务层 → 工具层。全链路流式（AsyncGenerator），数据从 API 流到 UI 无阻塞。

3. 查询循环 (Query Loop) CORE

4. 工具编排 (Tool Orchestration) TOOL

工具调用按 isConcurrencySafe 分区：并发安全的工具同批并行（最多 10 并发），非安全的串行执行。Context modifier 在批次结束后按序应用。

5. 流式工具执行器 (StreamingToolExecutor) TOOL

在 API 流式返回的同时执行工具。工具在 tool_use block 到达时立即入队执行，结果按接收顺序输出（不按完成顺序），Progress 消息实时推送。

sequenceDiagram
  participant API as Anthropic API
(Streaming)
  participant STE as StreamingToolExecutor
  participant TOOL as Tool Execution
  participant UI as REPL UI

  Note over API,UI: API 流式返回 + 工具并行执行

  API->>STE: stream: AssistantMessage (text)
  STE->>UI: yield message

  API->>STE: stream: tool_use block A (Read)
  STE->>STE: addTool(A) → status: queued
  STE->>STE: tryStartNext() → A concurrencySafe
  STE->>TOOL: execute A (async)
  Note over STE: A status: executing

  API->>STE: stream: tool_use block B (Grep)
  STE->>STE: addTool(B) → status: queued
  STE->>STE: tryStartNext() → B concurrencySafe
  STE->>TOOL: execute B (async, parallel)

  API->>STE: stream: tool_use block C (Bash)
  STE->>STE: addTool(C) → status: queued
  STE->>STE: tryStartNext() → C NOT safe, wait

  TOOL-->>STE: A progress update
  STE->>UI: yield progress (immediate)

  TOOL-->>STE: A complete → results buffered
  Note over STE: A status: completed

  STE->>STE: getCompletedResults()
  STE->>UI: yield A results (in order)
  Note over STE: A status: yielded

  TOOL-->>STE: B complete
  STE->>UI: yield B results
  Note over STE: B status: yielded

  STE->>STE: tryStartNext() → C can start now
  STE->>TOOL: execute C (exclusive)

  API->>STE: stream ends
  Note over STE: getRemainingResults()

  TOOL-->>STE: C complete
  STE->>UI: yield C results
