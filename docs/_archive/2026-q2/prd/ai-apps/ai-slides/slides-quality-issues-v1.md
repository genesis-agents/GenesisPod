## 一、背景

## 三、根因分析

### 3.1 技术根因

| 问题类型     | 根因                                      | 涉及模块                       |
| ------------ | ----------------------------------------- | ------------------------------ |
| 内边距不足   | 模板 CSS 未设置合适的 padding             | `templates/*.ts`               |
| 不可编辑     | HTML 渲染为图片作为背景，非原生 PPTX 元素 | `template-rendering.skill.ts`  |
| 图片渲染失败 | 图片获取失败时无降级处理                  | `image-fetcher.skill.ts`       |
| 无关内容重复 | 内容压缩时填充了默认占位内容              | `content-compression.skill.ts` |
| 缺少结构页   | 大纲规划未强制包含目录和结尾              | `outline-planning.skill.ts`    |

### 3.2 流程根因

| 问题类型     | 根因                           | 涉及模块                     |
| ------------ | ------------------------------ | ---------------------------- |
| 叙事跳跃     | 节奏控制未验证页面逻辑顺序     | `rhythm-controller.skill.ts` |
| 数据缺失     | 数据获取失败时未触发重试或降级 | `chart-renderer.skill.ts`    |
| 内容审核不足 | 质量审核未覆盖所有问题类型     | `quality-audit.skill.ts`     |

## 四、修复方案

### 4.1 P0-001: 页面内边距不足

**修复位置**: `backend/src/modules/ai/ai-office/slides/templates/`

**修复方案**:

1. 所有模板增加统一的安全边距 (padding: 60px 80px)
2. 创建全局样式变量控制边距
3. 确保内容不会触及页面边缘

**验收标准**:

- 所有页面内容距离边框至少 60px (上下) / 80px (左右)
- 在 100% 缩放下阅读体验良好

### 4.2 P0-002: 内容不可编辑

**调研结果**（2026-01-01 完成）:

系统已具备可编辑导出能力！关键发现：

| 模式            | 实现状态  | 文件位置                                                                 |
| --------------- | --------- | ------------------------------------------------------------------------ |
| 同源模式 (默认) | ✅ 已实现 | `slides-export.service.ts` - `renderSlidesFromHtml()`                    |
| 可编辑模式      | ✅ 已实现 | `slides-export.service.ts` - `renderSlide()` + `pptx-slides.renderer.ts` |

**当前问题**: `exportToPPTX()` 方法的 `editable` 参数默认为 `false`，导致始终使用 HTML 截图模式。

**修复方案（三选一）**:

1. **方案A - 默认可编辑**: 修改 `editable` 默认值为 `true`
   - 优点：用户立即获得可编辑 PPT
   - 缺点：视觉可能与预览有轻微差异

2. **方案B - 双版本导出**: 同时生成两个文件
   - `报告.pptx` - 同源高保真版本
   - `报告_editable.pptx` - 可编辑版本
   - 优点：满足所有场景
   - 缺点：文件体积翻倍

3. **方案C - 用户选择**: 在导出对话框添加选项
   - "高保真模式（与预览一致）"
   - "可编辑模式（文本可修改）"
   - 优点：用户自主选择
   - 缺点：需要前端改动

**推荐方案**: 方案A（短期）+ 方案C（长期）

**技术要点**:

- `PptxSlidesRenderer` 已支持15种模板的原生渲染
- 使用 `pptxgenjs` 库创建真实的 `<p:sp>` 形状元素
- 图表暂时使用占位符，需后续完善 `<c:chart>` 支持

**验收标准**:

- 用户可在 PowerPoint 中直接编辑所有文本
- 图表数据可修改（后续迭代）
- 样式保持基本一致

### 4.3 P0-003: 图片渲染失败

**修复位置**: `backend/src/modules/ai/ai-office/slides/skills/image-fetcher.skill.ts`

**修复方案**:

1. 图片获取失败时使用占位图或主题色块
2. 增加重试机制 (最多3次)
3. 记录失败日志便于排查
4. 提供默认的分类占位图库

**验收标准**:

- 无空白占位框出现
- 失败时显示美观的占位内容

### 4.4 P1-001: 无关内容重复

**修复位置**: `backend/src/modules/ai/ai-office/slides/skills/content-compression.skill.ts`

**修复方案**:

1. 移除默认的"创新驱动"占位内容
2. 内容不足时宁可减少条目，不填充无关内容
3. 添加语义相关性检查
4. 在 quality-audit 中添加内容相关性审核

**验收标准**:

- 每个条目与页面主题语义相关
- 无重复的无关填充内容

### 4.5 P1-002 ~ P1-006: 结构与叙事问题

**修复位置**:

- `outline-planning.skill.ts` - 大纲规划
- `rhythm-controller.skill.ts` - 节奏控制
- `narrative-planner.skill.ts` - 叙事规划

**修复方案**:

1. 强制生成目录页 (第2页) 和结尾页 (最后一页)
2. 验证页面逻辑顺序 (地理→人口→气候→...)
3. 封面页必须包含主题相关配图
4. 相关话题合并到同一章节
