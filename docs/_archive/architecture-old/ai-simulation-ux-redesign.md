# AI Simulation UX Redesign

## 问题分析

### 当前问题

1. **编辑场景界面混乱**：所有字段平铺在一个表单中，缺乏层次和引导
2. **布局不一致**：与系统其他页面（Data Collection）的布局风格不统一
3. **操作流程不清晰**：用户不知道先填什么后填什么
4. **缺乏默认值和智能提示**：用户需要从头填写所有字段

### 用户需求

- 快速创建推演场景
- 清晰的操作流程
- 合理的默认值
- 符合直觉的交互

## 设计方案

### 1. 统一布局风格

采用与Data Collection一致的布局结构：

```
+---------------------------+
| Sidebar | Header          |
|         |-----------------|
|         | Content         |
|         |                 |
+---------------------------+
```

- 去掉`max-w-6xl`限制，采用全宽布局
- 使用consistent padding: `px-8 py-6`
- 统一卡片样式和间距

### 2. 简化创建流程

**方案A：向导式创建（推荐）**

```
Step 1: 基本信息 (名称、行业、区域)
Step 2: 添加公司
Step 3: 配置角色
Step 4: 推演参数
```

**方案B：快速创建 + 高级编辑**

- 默认提供模板，一键创建
- 创建后可进入详情页编辑

### 3. 界面改进

#### 创建按钮

- 突出"从模板创建"选项
- 次要选项："从头创建"

#### 模板卡片

- 清晰展示模板内容预览
- 一键使用模板

#### 编辑界面

- 分tab展示：基本信息、公司、角色、参数
- 每个tab内容清晰分组
- 实时预览效果

### 4. 数据流程

```
用户点击"新建"
  ↓
选择模板 或 从头创建
  ↓
填写/确认基本信息
  ↓
添加公司（可选外部API同步）
  ↓
配置角色
  ↓
设置参数
  ↓
保存并可选择立即运行
```
