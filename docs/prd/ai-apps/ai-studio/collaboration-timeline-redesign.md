## 1. 问题分析

### 1.1 当前设计的核心问题

通过代码审查和用户反馈，当前「协作动态」面板存在以下问题：

| 问题类型           | 描述                                                           | 严重程度 |
| ------------------ | -------------------------------------------------------------- | -------- |
| **信息密度低**     | 每个 Agent 组仅显示名称 + "X 条活动记录"，需要展开才能看到内容 | 高       |
| **折叠设计反人性** | 用户需要逐个点击展开才能看到实际内容，增加操作成本             | 高       |
| **缺乏上下文**     | 无法一眼看出研究的整体进度和当前阶段                           | 中       |
| **重要信息被埋没** | 关键发现、研究结论等重要内容被折叠在树形结构深处               | 高       |
| **状态表达模糊**   | "已完成" 状态缺乏具体含义，不知道完成了什么                    | 中       |
| **时间线断裂**     | 按 Agent 分组破坏了时间连续性，无法理解事件顺序                | 中       |

### 1.2 当前数据结构分析

```typescript
// 当前 UI 消息结构
interface UIMessage {
  id: string;
  type: "leader" | "agent" | "progress" | "system";
  agent: string; // Agent 名称
  agentIcon: string; // 图标
  agentType?: string; // leader/researcher/reviewer/synthesizer
  content: string; // 消息内容（通常很短）
  timestamp: Date;
  detail?: MessageDetail; // 详情（需要展开）
  progress?: number; // 进度百分比
}

// 当前分组结构
interface AgentGroup {
  groupKey: string; // 分组键
  agentName: string; // Agent 名称
  messages: UIMessage[]; // 消息列表
  isCompleted: boolean; // 是否完成
}
```

**问题**：数据结构本身是合理的，但 UI 展示方式将有价值的 `content` 和 `detail` 隐藏了。

### 1.3 用户场景分析

用户在查看协作动态时，实际想知道的是：

1. **当前谁在做什么？**（实时状态）
2. **研究发现了什么？**（关键结论）
3. **整体进度如何？**（宏观把控）
4. **有没有问题需要我关注？**（异常告警）

当前设计完全没有满足这些需求。

## 2. 设计目标

### 2.1 核心目标

| 目标             | 描述                             | 衡量标准                       |
| ---------------- | -------------------------------- | ------------------------------ |
| **信息一目了然** | 不需要任何点击，关键信息直接可见 | 用户 3 秒内能理解当前状态      |
| **时间线连贯**   | 保持事件的时间顺序，支持实时追踪 | 新消息出现时用户能立即感知     |
| **突出重点**     | 研究发现、关键决策等内容视觉突出 | 重要信息占据视觉焦点           |
| **渐进详情**     | 支持查看更多细节，但不强制展开   | 默认显示摘要，hover/点击看全文 |

### 2.2 非目标

- 不改变底层数据结构
- 不修改 WebSocket 事件格式
- 不影响现有功能（Leader 计划显示等）

## 3. 新方案设计

### 3.1 设计理念：消息流 + 智能卡片

参考业界最佳实践：

- **GitHub Activity Feed**：时间线 + 内容摘要 + 可展开详情
- **Slack 消息流**：按时间顺序、发送者头像 + 消息内容
- **Linear 活动日志**：紧凑的时间线 + 语义化的活动描述

**核心变化**：从"按 Agent 分组的折叠树"改为"按时间排序的智能消息流"

## 4. 关键交互说明

### 4.1 实时更新动效

| 事件       | 动效                             |
| ---------- | -------------------------------- |
| 新消息到达 | 卡片从底部滑入，配合轻微闪烁高亮 |
| 进度更新   | 进度条平滑动画                   |
| 状态变更   | 状态徽章颜色渐变过渡             |
| 研究完成   | 卡片边框短暂高亮绿色             |

### 4.2 展开/收起逻辑

- **默认展开**：关键发现、审核结论、报告摘要
- **默认收起**：详细内容、完整引用列表
- **智能收起**：当消息超过 5 条时，自动收起早期消息（保留最近 3 条展开）

### 4.3 筛选能力

顶部添加简单筛选：

```
[全部] [Leader] [研究员] [审核员] [撰写员]
```

点击可筛选显示特定角色的消息。

### 4.4 消息跳转

- 点击维度名称：跳转到对应维度的研究内容 Tab
- 点击"查看完整报告"：跳转到报告 Tab
- 点击引用来源：展开引用列表

在 `uiMessages` 基础上添加一层智能处理：

```typescript

  // 从 detail 中提取的关键信息
  keyFindings?: string[];
  summary?: string;
  citationCount?: number;
  reviewResult?: {
    completeness: string;
    reliability: string;
    coherence: string;
    passed: boolean;
  };
  // 显示控制
  isCollapsed: boolean;
  canCollapse: boolean;
}

function enrichMessages(messages: UIMessage[]): EnrichedMessage[] {
  return messages.map((msg) => {
    // 从 detail 中提取关键信息到顶层
    // 设置默认展开/收起状态
  });
}
```

### 5.2 组件拆分

```
TeamInteractionTabContent
├── ProgressOverview         // 进度概览
├── MessageFilter            // 筛选栏
└── MessageStream            // 消息流
    ├── LeaderPlanCard       // Leader 规划卡片
    ├── ResearchCompleteCard // 研究完成卡片（带发现）
    ├── ResearchProgressCard // 研究进行中卡片
    ├── ReviewCard           // 审核结果卡片
    └── ReportCard           // 报告完成卡片
```

### 5.3 性能考虑

- 使用虚拟滚动处理大量消息
- 消息卡片使用 `React.memo` 优化
- 进度动画使用 CSS transition，避免 JS 动画

### 5.4 移除的功能

- 移除按 Agent 分组的折叠树视图
- 移除 `expandedGroups` 状态
- 移除 `toggleGroupExpand` 方法
