## 1. 概述

### 1.1 背景

在 AI Research 模块中，用户会生成深度研究报告（Insights Report）。这些报告包含大量的研究发现、分析和结论。为了支持用户在报告上进行标注、评论和协作审阅，需要提供完整的批注功能。

批注功能使用户能够：

- 标注报告中的重要内容
- 记录分析思路和问题
- 与团队成员进行协作讨论
- 沉淀研究过程中的知识

### 1.2 目标

1. **提升研究效率**: 让用户快速标记和回顾重要发现
2. **支持团队协作**: 实现多人审阅和讨论
3. **知识沉淀**: 将研究过程中的思考和发现结构化保存
4. **无缝体验**: 批注操作不干扰阅读流程

### 1.3 非目标（本期不做）

- 批注导出为独立文档
- 批注模板和预设标签
- 批注的 AI 智能总结
- 跨报告的批注关联
- 批注的版本对比
- 离线批注同步

## 2. 功能定义

### 2.1 核心功能列表

| ID    | 功能名称     | 描述                     | 优先级 |
| ----- | ------------ | ------------------------ | ------ |
| F-001 | 文本选择批注 | 选中报告文本后添加批注   | P0     |
| F-002 | 批注高亮显示 | 已批注文本显示高亮背景   | P0     |
| F-003 | 批注颜色分类 | 支持5种颜色区分批注类型  | P0     |
| F-004 | 批注列表面板 | 侧边栏显示所有批注       | P0     |
| F-005 | 批注跳转定位 | 点击批注跳转到对应位置   | P0     |
| F-006 | 批注编辑删除 | 修改或删除自己的批注     | P0     |
| F-007 | 批注状态管理 | 待处理/已解决/已归档     | P1     |
| F-008 | 批注回复     | 对批注进行回复讨论       | P1     |
| F-009 | 批注筛选     | 按状态、颜色、作者筛选   | P1     |
| F-010 | 跨段落批注   | 支持选中跨多个段落的文本 | P1     |

### 2.2 功能边界

#### 做什么

- 纯文本内容的批注
- Markdown 渲染后内容的批注
- 报告正文区域的批注
- 实时保存和同步

#### 不做什么

- 图片、表格的批注
- PDF/Word 导出后的批注
- 代码块内部的批注
- 批注的版本历史

### 2.3 与其他功能的关系

```
+----------------+     +-----------------+     +------------------+
|  洞察报告      |---->|   批注功能      |<----|   团队协作       |
| (Insights     |     | (Annotation)    |     | (Collaboration)  |
|  Report)      |     |                 |     |                  |
+----------------+     +-----------------+     +------------------+
       |                      |                       |
       v                      v                       v
+----------------+     +-----------------+     +------------------+
|  引用管理      |     |   AI 编辑       |     |   版本历史       |
| (References)  |     | (AI Edit)       |     | (Revisions)      |
+----------------+     +-----------------+     +------------------+
```

- **洞察报告**: 批注的载体，批注附着在报告内容上
- **团队协作**: 批注是协作审阅的核心工具
- **引用管理**: 批注可标注需要验证的引用
- **AI 编辑**: 选中文本可以选择添加批注或 AI 编辑
- **版本历史**: 报告版本变更后批注需要重新定位

## 3. 用户场景用例（User Stories）

### 3.1 研究员角色

#### US-001: 标注重要发现

```
作为 研究员，
我想要 在报告中标注关键发现，
以便 后续快速回顾和引用。

验收标准：
- 可以选中任意文本添加批注
- 可以选择颜色区分重要程度
- 批注自动保存不丢失
```

#### US-002: 记录分析思路

```
作为 研究员，
我想要 在报告旁边记录我的分析思考，
以便 记录研究过程和推理逻辑。

验收标准：
- 批注支持输入详细内容
- 可以编辑已有批注
- 批注内容支持基本格式
```

#### US-003: 标记待验证内容

```
作为 研究员，
我想要 标记需要进一步验证的内容，
以便 后续针对性地查证。

验收标准：
- 可以用特定颜色标记待验证内容
- 验证完成后可以标记为已解决
- 可以筛选出所有待验证项
```

### 3.2 团队协作角色

#### US-004: 审阅反馈

```
作为 团队审阅者，
我想要 在报告上添加修改建议，
以便 作者了解改进方向。

验收标准：
- 可以对他人报告添加批注
- 批注显示作者信息
- 作者可以看到所有批注
```

#### US-005: 讨论交流

```
作为 团队成员，
我想要 对批注进行回复讨论，
以便 深入交流观点。

验收标准：
- 可以回复任何批注
- 回复形成讨论线程
- 新回复有提示通知
```

#### US-006: 处理反馈

```
作为 报告作者，
我想要 标记已处理的批注，
以便 跟踪反馈处理进度。

验收标准：
- 可以将批注标记为已解决
- 已解决批注视觉上有区分
- 可以筛选待处理批注
```

### 3.3 知识沉淀角色

#### US-007: 回顾批注

```
作为 知识管理者，
我想要 浏览所有批注内容，
以便 沉淀有价值的分析。

验收标准：
- 批注列表按时间排序
- 可以按作者、颜色筛选
- 可以看到批注上下文
```

#### US-008: 批注导航

```
作为 用户，
我想要 在批注和原文间快速跳转，
以便 高效查阅。

验收标准：
- 点击批注跳转到原文位置
- 点击高亮文本显示批注详情
- 滚动平滑有动画
```

## 4. 用户旅程（User Journey）

### 4.1 创建批注流程

```
+-------------+     +---------------+     +----------------+     +-------------+
|  阅读报告   |---->| 选中目标文本  |---->| 显示上下文菜单 |---->| 选择批注    |
|             |     |               |     |                |     |             |
+-------------+     +---------------+     +----------------+     +-------------+
                                                                       |
                                                                       v
+-------------+     +---------------+     +----------------+     +-------------+
|  保存成功   |<----| 点击确认      |<----| 输入批注内容   |<----| 选择颜色    |
|  显示高亮   |     |               |     |                |     |             |
+-------------+     +---------------+     +----------------+     +-------------+
```

**详细步骤**:

1. **阅读报告**: 用户在洞察报告页面浏览内容
2. **选中文本**: 使用鼠标拖选需要批注的文本
3. **触发菜单**: 松开鼠标后自动显示上下文菜单
4. **选择批注**: 点击"添加批注"选项
5. **选择颜色**: 从5种颜色中选择一种
6. **输入内容**: 在弹出的输入框中输入批注内容
7. **确认保存**: 点击保存按钮
8. **显示结果**: 选中文本显示高亮，批注出现在侧边栏

### 4.2 查看批注流程

```
+----------------+     +------------------+
|  方式1:        |     |  方式2:          |
|  点击高亮文本  |     |  点击侧边栏批注  |
+----------------+     +------------------+
        |                      |
        v                      v
+----------------+     +------------------+
|  高亮变为选中  |     |  原文滚动定位    |
|  态（带边框） |     |  高亮闪烁提示    |
+----------------+     +------------------+
        |                      |
        +----------+-----------+
                   |
                   v
          +----------------+
          |  显示批注详情  |
          |  - 作者信息    |
          |  - 批注内容    |
          |  - 操作按钮    |
          +----------------+
```

### 4.3 跳转定位流程

--------                      --------
点击侧边栏批注 ---------> 定位到报告中对应位置

## 5. 用户体验设计要点

### 5.1 交互设计原则

#### 5.1.1 即时反馈

- 选中文本立即显示上下文菜单（<100ms）
- 添加批注后立即显示高亮（<200ms）
- 操作成功/失败有明确提示

#### 5.1.2 最小干扰

- 上下文菜单紧贴选中区域
- 菜单不遮挡选中文本
- 点击外部自动关闭菜单

#### 5.1.3 一致性

- 颜色语义保持一致
- 操作入口位置固定
- 动画风格统一

#### 5.1.4 可发现性

- 首次使用提供引导
- 按钮配有文字说明
- 悬停显示操作提示

### 5.2 视觉反馈机制

#### 5.2.1 高亮颜色系统

| 颜色 | 色值          | 语义建议  | 使用场景       |
| ---- | ------------- | --------- | -------------- |
| 黄色 | bg-yellow-200 | 普通标注  | 一般重要内容   |
| 绿色 | bg-green-200  | 确认/支持 | 已验证的发现   |
| 蓝色 | bg-blue-200   | 问题/疑问 | 需要讨论的点   |
| 粉色 | bg-pink-200   | 重要/优先 | 关键发现       |
| 紫色 | bg-purple-200 | 待验证    | 需要核实的内容 |

#### 5.2.2 状态视觉

| 状态               | 视觉表现                   |
| ------------------ | -------------------------- |
| 活跃 (active)      | 正常高亮颜色               |
| 已解决 (resolved)  | 高亮变淡 + "已解决"标签    |
| 已归档 (archived)  | 隐藏高亮（可通过筛选查看） |
| 选中 (highlighted) | 蓝色边框 + 脉冲动画        |

#### 5.2.3 动画效果

```css
/* 跳转时的脉冲动画 */
@keyframes annotation-pulse {
  0%,
  100% {
    box-shadow: 0 0 0 0 rgba(59, 130, 246, 0.5);
  }
  50% {
    box-shadow: 0 0 0 4px rgba(59, 130, 246, 0.5);
  }
}

.annotation-pulse {
  animation: annotation-pulse 0.5s ease-in-out 3;
}
```

### 5.3 错误处理和边界情况

#### 5.3.1 错误场景处理

| 错误场景     | 处理方式                         |
| ------------ | -------------------------------- |
| 网络失败保存 | Toast 提示 + 本地暂存 + 自动重试 |
| 内容被删除   | 提示"原文已变更" + 保留批注内容  |
| 权限不足     | 提示"无权操作" + 隐藏按钮        |
| 批注定位失败 | 提示"无法定位" + 显示批注内容    |

#### 5.3.2 边界情况

| 边界情况                | 处理方式                 |
| ----------------------- | ------------------------ |
| 选中内容过短（<5字）    | 允许添加，但给予提示     |
| 选中内容过长（>1000字） | 显示前100字 + "..."      |
| 跨段落选择              | 支持，使用多节点高亮     |
| 报告内容更新            | 尝试重新定位，失败则标记 |
| 重复位置批注            | 允许多个批注叠加         |

### 5.4 可访问性考虑

#### 5.4.1 键盘操作

- Tab 键在批注间导航
- Enter 键展开/收起批注
- Escape 键关闭弹出层
- Ctrl+Shift+A 快捷添加批注

#### 5.4.2 屏幕阅读器

- 高亮区域有 aria-label 描述
- 批注操作按钮有清晰标签
- 状态变化有语音提示

#### 5.4.3 视觉辅助

- 高对比度模式支持
- 色盲友好的辅助标识
- 足够的文字大小（最小14px）

## 8. 技术约束和风险

### 8.1 已知技术限制

#### 8.1.1 DOM 定位精度

**问题**: 批注依赖文本位置定位，但 DOM 结构可能变化

**当前方案**:

- 使用 `TextSelector` 结构存储 exact + prefix + suffix
- 通过上下文匹配增加定位准确性
- 支持模糊匹配（去除引用标记后匹配）

**代码位置**: `frontend/lib/annotation/text-finder.ts`

```typescript
interface TextSelector {
  exact: string; // 选中的文本
  prefix?: string; // 前50字上下文
  suffix?: string; // 后50字上下文
}
```

#### 8.1.2 跨段落高亮

**问题**: DOM Range 跨元素时无法直接使用 surroundContents

**当前方案**:

- 遍历 Range 内所有文本节点
- 逆序处理避免 DOM 变化影响
- 每个文本节点单独包裹 mark 元素

**代码位置**: `frontend/lib/annotation/dom-highlighter.ts`

```typescript
// 逆序处理文本节点，避免 DOM mutation 问题
for (let i = textNodeInfos.length - 1; i >= 0; i--) {
  const mark = wrapTextNodePortion(info.node, ...);
}
```

#### 8.1.3 React 协调冲突 (已解决)

**问题**: 添加批注时崩溃 `"Failed to execute 'insertBefore' on 'Node'"`

**根因分析**:

1. `AnnotationHighlighter` 通过直接操作 DOM 添加 `<mark>` 元素
2. 当 `annotations` 状态更新时，React 重新渲染 `ReportEditor`
3. React 尝试协调 (reconcile) `ReactMarkdown` 的虚拟 DOM 与实际 DOM
4. 实际 DOM 已被 `<mark>` 元素修改，与虚拟 DOM 不匹配
5. React 协调失败，抛出 `insertBefore` 错误

**解决方案** (2026-01-16):

在 `ReportEditor.tsx` 中使用 `useMemo` 缓存 ReactMarkdown 的 JSX 元素：

```typescript
const memoizedMarkdownContent = useMemo(
  () => (
    <article className="prose prose-gray max-w-none">
      <ReactMarkdown ...>
        {markdownContent}
      </ReactMarkdown>
    </article>
  ),
  [markdownContent, processText] // 不依赖 annotations
);
```

**工作原理**:

1. 当 annotations 改变时，ReportEditor 重新渲染
2. `useMemo` 返回相同的 JSX 元素引用（因为 markdownContent 未变）
3. React 检测到相同引用，跳过子元素的协调过程
4. `AnnotationHighlighter` 的 effect 安全地修改 DOM

**代码位置**: `frontend/components/ai-research/ReportEditor.tsx:982-1050`

#### 8.1.4 引用标记干扰

**问题**: 报告中的引用标记 [1] 可能影响文本匹配

**当前方案**:

- 匹配时移除引用标记
- 支持多种引用格式 `[1]`, `[1,2]`, `__CITE_GROUP_1__`

```typescript
function removeCitationMarkers(text: string): string {
  return text
    .replace(/\[(\d+(?:\s*[,、]\s*\d+)*)\]/g, "")
    .replace(/__CITE_GROUP_[\d_]+__/g, "");
}
```
