# AI Research Tab 系统梳理计划

## 一、问题诊断

### 当前Tab结构（6个）

| Tab | 名称       | 核心功能        | 问题                      |
| --- | ---------- | --------------- | ------------------------- |
| 1   | TODO LIST  | 任务列表 + 进度 | 数据来源与左侧面板相同    |
| 2   | Agent 协作 | WebSocket消息流 | 与TODO LIST定位重叠       |
| 3   | 洞察报告   | 报告内容展示    | 无问题                    |
| 4   | 研究历史   | 完整时间线      | 功能过重，与Agent协作重叠 |
| 5   | 可信度     | 质量评估        | 无问题                    |
| 6   | 参考文献   | 引用来源        | 无问题                    |

### 核心矛盾

1. **TODO LIST vs Agent 协作**：两者都在展示"研究进展"，定位模糊
2. **Agent 协作 vs 研究历史**：都展示Agent活动，视角不同但内容重叠
3. **信息分散**：用户需要在多个Tab间切换才能获得完整信息

---

## 二、PRD原始设计意图

PRD的核心思路是：**5个Tab，明确分工**

| Tab          | 定位               | 用户场景             |
| ------------ | ------------------ | -------------------- |
| **研究协作** | 实时进度中枢       | 研究进行中，掌控全局 |
| **洞察报告** | 最终成果           | 研究完成后，阅读报告 |
| **研究历史** | 历史追溯（简化版） | 事后复盘，版本对比   |
| **可信度**   | 质量验证           | 验证来源可靠性       |
| **参考文献** | 引用管理           | 查看完整引用列表     |

**关键设计**：Agent思考过程整合到"研究协作"Tab的TODO详情展开中

## 四、各Tab边界定义

### 信息归属矩阵

| 信息类型       | TODO LIST | Agent思考 | 洞察报告 | 研究历史 | 可信度  | 参考文献 |
| -------------- | :-------: | :-------: | :------: | :------: | :-----: | :------: |
| 任务进度条     |  ✅ 主要  |     -     |    -     |    -     |    -    |    -     |
| 任务列表状态   |  ✅ 主要  |     -     |    -     |    -     |    -    |    -     |
| 快捷指令/输入  |  ✅ 主要  |     -     |    -     |    -     |    -    |    -     |
| Agent思考链路  | 展开详情  |  ✅ 主要  |    -     |    -     |    -    |    -     |
| Leader决策过程 | 展开详情  |  ✅ 主要  |    -     |    -     |    -    |    -     |
| 完整报告内容   |     -     |     -     | ✅ 主要  |    -     |    -    |    -     |
| 历史会话列表   |     -     |     -     |    -     | ✅ 主要  |    -    |    -     |
| 版本对比功能   |     -     |     -     |    -     | ✅ 主要  |    -    |    -     |
| 来源质量评分   |     -     |     -     |    -     |    -     | ✅ 主要 |    -     |
| 引用列表管理   |     -     |     -     |    -     |    -     |    -    | ✅ 主要  |

### 核心差异说明

**TODO LIST vs Agent思考**

```
TODO LIST (任务视角):
"技术趋势维度研究" - 进行中 65% - [暂停] [查看详情]

Agent思考 (思考视角):
10:31:00 🔍 研究员开始搜索技术趋势相关资料
10:31:15    → 搜索策略: 学术论文 + 行业报告 + 新闻动态
10:31:30    → 找到42条结果，过滤后保留15条高质量来源
10:32:00    → 开始分析主要论点...
10:32:45    → 发现关键洞察: AI芯片市场预计2025年达到xxx亿
```

**Agent思考 vs 研究历史**

```
Agent思考 (当前研究):
显示本次研究中所有Agent的实时活动

研究历史 (所有研究):
第1次研究 → 第2次研究 → 第3次研究 → ... → 当前研究
每次研究是一个"快照"，可以对比和回滚
```

### 各Tab详细用户旅程

#### Tab 1: TODO LIST（研究协作）

**用户角色**: 研究进度的"监控者"和"指挥者"

**进入场景**:

- 刚启动研究，想看任务分解
- 研究进行中，想了解当前进度
- 想发送指令调整研究方向

**用户旅程**:

```
┌─────────────────────────────────────────────────────────────┐
│ 用户进入 TODO LIST Tab                                      │
│                                                             │
│ Step 1: 一眼看全局                                          │
│ ────────────────────────────────────────────────────────── │
│ 用户看到: 进度条 75% | 预计剩余3分钟                        │
│ 用户心理: "还有1/4没完成，大概还要等3分钟"                  │
│                                                             │
│ Step 2: 了解任务分布                                        │
│ ────────────────────────────────────────────────────────── │
│ 用户看到:                                                   │
│   ⏳ 进行中 (2): 技术趋势研究 65%、市场格局研究 40%         │
│   ⏸ 待处理 (3): 投资动态、报告撰写、质量审核               │
│   ✅ 已完成 (4): Leader规划、政策法规、人才教育、国际竞争   │
│ 用户心理: "原来还有3个任务没开始，正在做2个"                │
│                                                             │
│ Step 3: 查看任务详情（可选）                                │
│ ────────────────────────────────────────────────────────── │
│ 用户操作: 点击"技术趋势研究"任务                            │
│ 系统响应: 右侧展开详情面板                                  │
│ 用户看到:                                                   │
│   - 执行者: 研究员-技术方向                                 │
│   - 开始时间: 10:31:00                                      │
│   - 当前状态: 正在分析论文数据...                           │
│   - Agent活动记录（简要版）                                 │
│                                                             │
│ Step 4: 发送指令（可选）                                    │
│ ────────────────────────────────────────────────────────── │
│ 场景A: 用户点击快捷指令 [深入分析]                          │
│ 场景B: 用户输入 "@Leader 请重点关注AI芯片领域"              │
│ 系统响应:                                                   │
│   - 新增TODO: "用户请求: 深入分析AI芯片"                    │
│   - Leader开始理解和执行                                    │
│                                                             │
│ Step 5: 任务操作（可选）                                    │
│ ────────────────────────────────────────────────────────── │
│ 用户操作: 点击某个任务的 [暂停] 按钮                        │
│ 系统响应: 任务状态变为"已暂停"                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**离开场景**:

- 进度100% → 切换到"洞察报告"Tab
- 想了解AI思考细节 → 切换到"Agent思考"Tab
- 任务完成 → 关闭页面

## 七、最终确定方案

### 确定的Tab结构（6个Tab）

| 顺序 | Tab名称       | 定位               | 用户角色      |
| ---- | ------------- | ------------------ | ------------- |
| 1    | **TODO LIST** | 任务管理中枢       | 监控者/指挥者 |
| 2    | **协作动态**  | AI思考链路深度视图 | 观察者        |
| 3    | **洞察报告**  | 最终研究成果       | 读者/编辑者   |
| 4    | **研究历史**  | 历史版本档案       | 回顾者        |
| 5    | **可信度**    | 数据质量评估       | 审查者        |
| 6    | **参考文献**  | 引用来源管理       | 追溯者        |

### Tab核心区别总结

```
TODO LIST vs 协作动态
─────────────────────────────────────────────────
TODO LIST:  结构化任务视图 - "做什么、进度多少"
协作动态:   详细思考链路 - "怎么想的、为什么这样做"

协作动态 vs 研究历史
─────────────────────────────────────────────────
协作动态:   当前研究的实时思考过程
研究历史:   所有历史研究的会话记录和版本对比
```
