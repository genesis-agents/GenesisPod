## 文档诊断分析

### 发现的问题

#### 1. 目录结构不一致 (严重)

**问题描述**:

- 索引中记录的路径: `ai-studio/`, `ai-office/`, `ai-group/`
- 实际文件路径: `ai-apps/ai-studio/`, `ai-apps/ai-office/`, `ai-teams/`
- 缺少顶级分类: ai-apps, ai-teams, infra

**影响**:

- 开发者无法通过索引找到文档
- 文档可信度降低

**解决方案**: 创建清晰的三层结构，明确分类

#### 2. 命名不统一 (中等)

**问题描述**:

- 文档中称为 "AI Group"
- 代码中是 `ai-teams` 模块
- 产品功能是 "AI Teams 协作系统"

**影响**:

- 团队内部沟通术语混乱
- 新成员理解困难

**解决方案**: 统一使用 "AI Teams"，在索引中注明对应关系

#### 3. 版本管理混乱 (中等)

**问题描述**:

- 多个版本文档并存 (v1.0, v2.0, v3.0, v3.1, v4.0)
- 无明确的"当前版本"标识
- 归档文档与有效文档混在一起

**影响**:

- 不知道该参考哪个版本
- 历史版本占用主目录空间

**解决方案**: 分离 current/ 和 archive/，明确标识当前版本

#### 4. 遗失模块 (低)

**发现**:

- `ai-writing` 模块有 4 个 PRD 文档，但原索引中未提及
- `ai-studio` 有 3 个子文档未在索引中

**影响**:

- 文档不完整

**解决方案**: 在新索引中补全

## 新文档结构说明

### 目录层次

```
docs/prd/
├── current/              ← 当前有效 PRD
│   ├── ai-apps/          # AI 应用层
│   │   ├── ai-studio/
│   │   ├── ai-office/
│   │   ├── ai-slides/
│   │   ├── ai-coding/
│   │   ├── ai-ask/
│   │   └── ai-writing/
│   ├── ai-teams/         # AI Teams 协作系统
│   │   ├── [核心文档]
│   │   └── topic-research/
│   └── infra/            # 基础设施层
│       ├── core/
│       ├── knowledge-base/
│       ├── library/
│       ├── integrations/
│       └── data-collection/
├── archive/              ← 历史版本归档
│   └── [同上结构]
├── readme.md             # 主索引（向后兼容）
├── current/readme.md     # 当前 PRD 详细索引
├── archive/readme.md     # 归档文档索引
├── prd-cleanup-plan.md   # 整理计划
├── prd-cleanup-report.md # 本报告
└── prd-renaming-map.md   # 重命名映射
```

### 设计原则

1. **清晰分层**: 按架构分层组织 (AI Apps, AI Teams, Infra)
2. **版本分离**: current/ 只保留当前版本，archive/ 存放历史版本
3. **向后兼容**: 主 readme.md 保留常用链接，方便过渡
4. **术语统一**: 在索引中明确标注术语对应关系

## 文档质量改进

### 完善的索引系统

**主索引 (readme.md)**:

- 提供快速导航到 current/ 和 archive/
- 包含模块状态总览表
- 明确术语对应关系
- 向后兼容常用链接

**当前 PRD 索引 (current/readme.md)**:

- 按三层架构组织
- 列出每个模块的主 PRD 和子文档
- 标注版本、状态、说明
- 提供文档更新流程指南

**归档索引 (archive/readme.md)**:

- 列出所有归档文档
- 说明归档原因和日期
- 提供恢复指南

### 命名规范

**统一标准**:

- 小写字母 + 连字符 (kebab-case)
- 格式: `[模块]-[类型]-v[版本].md`
- 示例: `ai-teams-prd-v1.0.md`

**禁止项**:

- ❌ 大写字母
- ❌ 下划线
- ❌ 中文文件名
- ❌ 缺少版本号

### 文档维护流程

**新建文档**:

1. 在 `current/` 对应目录创建
2. 遵循命名规范
3. 更新索引文件

**更新文档**:

- 小更新: 直接修改，递增版本号
- 大更新: 移动旧版本到 `archive/`，创建新版本

**废弃文档**:

1. 移至 `archive/`
2. 添加 `-archived` 后缀
3. 在归档索引中说明原因
