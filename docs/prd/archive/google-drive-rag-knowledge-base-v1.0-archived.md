## 1. Executive Summary

### 1.1 Background

DeepDive Engine 已实现 Google Drive 基础集成（连接、导入、导出），用户可以将云端文档导入到资源库。然而，当前导入的内容仅作为静态资源存储，无法被 AI 系统智能检索和利用。

用户的核心诉求是：**将 Google Drive 中的专业文档（如行业报告、内部知识库、研究资料）作为 AI 的领域知识输入，使后续的 AI Ask（智能问答）和 AI Studio（深度研究）能够基于这些私有知识生成更准确、更专业的回答和报告。**

这是 RAG（Retrieval-Augmented Generation）架构的典型应用场景：

- **Retrieval**：从用户的 Google Drive 文档中检索相关信息
- **Augmented**：将检索到的内容作为上下文增强 AI 的回答
- **Generation**：基于增强后的上下文生成高质量输出

### 1.2 Problem Statement

| 问题                   | 影响                                          | 当前状态 |
| ---------------------- | --------------------------------------------- | -------- |
| AI 缺乏领域知识        | AI 只能基于通用知识回答，无法利用用户私有文档 | 未解决   |
| 文档检索效率低         | 用户需要手动查找相关文档，再复制粘贴给 AI     | 体验差   |
| 知识更新不及时         | 新导入的文档无法自动被 AI 利用                | 未解决   |
| 多文档综合分析困难     | AI 无法跨多个文档进行信息整合                 | 部分支持 |
| 深度研究缺乏专业数据源 | AI Studio 只能搜索公开网络信息                | 未解决   |

### 1.3 Goals

| Goal           | Description                                    | Success Metric     |
| -------------- | ---------------------------------------------- | ------------------ |
| **知识库构建** | 用户可以将 Google Drive 文档构建为私有知识库   | 向量化成功率 > 95% |
| **智能检索**   | AI 能够自动检索知识库中的相关内容              | 检索相关度 > 80%   |
| **上下文增强** | AI Ask 和 AI Studio 能够利用知识库增强回答质量 | 用户满意度提升 30% |
| **实时同步**   | Google Drive 文档更新后自动同步到知识库        | 同步延迟 < 5 分钟  |
| **透明引用**   | AI 回答能够标注来源文档，支持溯源验证          | 引用准确率 > 95%   |

### 1.4 Non-Goals

- **实时协作编辑**：不在 DeepDive 内直接编辑 Google Drive 文档
- **全量文档索引**：不自动索引用户所有 Google Drive 内容，需用户明确选择
- **跨用户知识共享**：本期不支持将个人知识库共享给其他用户
- **非文本内容**：不支持图片、视频、音频的语义理解（仅支持文本提取）
- **多语言混合**：不支持单个文档中多种语言的分块优化

#### 3.3.3 智能分块策略

| 策略             | Description                         | 适用场景                   |
| ---------------- | ----------------------------------- | -------------------------- |
| **语义分块**     | 按段落/章节边界分块，保持语义完整性 | 结构化文档（报告、论文）   |
| **固定大小分块** | 按固定 token 数分块，带重叠         | 长文本、无明确结构的文档   |
| **递归分块**     | 先按大结构分，再按小结构细分        | 复杂文档（书籍、技术手册） |
| **句子分块**     | 按句子分块，适用于 FAQ 类内容       | Q&A 文档、FAQ              |

#### 3.3.4 分块参数配置

```typescript
interface ChunkingConfig {
  strategy: "semantic" | "fixed" | "recursive" | "sentence";

  // 固定大小分块参数
  chunkSize: number; // 目标 token 数 (default: 512)
  chunkOverlap: number; // 重叠 token 数 (default: 50)

  // 语义分块参数
  separators: string[]; // 分隔符优先级 ['\n\n', '\n', '. ', ' ']

  // 元数据保留
  preserveMetadata: boolean; // 保留文档标题、章节信息
  addPosition: boolean; // 添加位置信息 (页码/章节)
}
```

#### 3.3.5 Embedding 模型选择

| 模型                       | 维度 | 成本         | 性能 | 推荐场景           |
| -------------------------- | ---- | ------------ | ---- | ------------------ |
| **text-embedding-3-small** | 1536 | $0.02/1M tok | 良好 | 默认选择（性价比） |
| **text-embedding-3-large** | 3072 | $0.13/1M tok | 优秀 | 高精度需求         |
| **text-embedding-ada-002** | 1536 | $0.10/1M tok | 良好 | 兼容性需求         |

#### 3.3.6 Acceptance Criteria

- [ ] 支持 PDF、DOCX、DOC、TXT、MD 格式的内容提取
- [ ] 支持 Google Docs/Sheets/Slides 格式（通过 Google API 导出）
- [ ] 默认使用语义分块，保持段落完整性
- [ ] 每个 chunk 包含：内容、位置信息、父文档引用
- [ ] 向量化使用 text-embedding-3-small 模型
- [ ] 处理失败时记录错误，不影响其他文档

### 3.6 F-005: 同步与更新机制

#### 3.6.1 Description

保持知识库与 Google Drive 源文件的同步，支持增量更新和版本管理。

#### 3.6.2 同步策略

| 策略         | 触发条件                     | 行为                       |
| ------------ | ---------------------------- | -------------------------- |
| **手动同步** | 用户点击"同步"按钮           | 检查所有文档更新，增量索引 |
| **自动同步** | 定时任务（可配置间隔）       | 后台检查更新，静默索引     |
| **实时同步** | Google Drive Webhook（未来） | 即时检测变更，立即索引     |
| **导入同步** | 用户导入新文档到 Library     | 询问是否加入现有知识库     |

// 知识库
model KnowledgeBase {
  id          String   @id @default(uuid())
  userId      String   @map("user_id")
  name        String
  description String?

  // 状态
  status      KnowledgeBaseStatus @default(CREATING)
  lastError   String?  @map("last_error")

  // 统计
  documentCount Int    @default(0) @map("document_count")
  chunkCount    Int    @default(0) @map("chunk_count")
  totalTokens   Int    @default(0) @map("total_tokens")

  // 时间戳
  createdAt   DateTime @default(now()) @map("created_at")
  updatedAt   DateTime @updatedAt @map("updated_at")
  lastSyncAt  DateTime? @map("last_sync_at")

  // 关系
  user        User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  documents   KnowledgeBaseDocument[]

  @@index([userId])
  @@map("knowledge_bases")
}

  CREATING
  INDEXING
  READY
  SYNCING
  ERROR
}

// 知识库文档
model KnowledgeBaseDocument {
  id              String   @id @default(uuid())
  knowledgeBaseId String   @map("knowledge_base_id")

  // 来源信息
  sourceType      DocumentSourceType @default(GOOGLE_DRIVE)
  sourceId        String   @map("source_id")  // Google Drive file ID 或 Resource ID

  // 文档信息
  title           String
  mimeType        String   @map("mime_type")
  fileSize        Int?     @map("file_size")

  // 索引状态
  indexStatus     DocumentIndexStatus @default(PENDING)
  indexError      String?  @map("index_error")
  chunkCount      Int      @default(0) @map("chunk_count")

  sourceModifiedAt DateTime? @map("source_modified_at")

  // 时间戳
  createdAt       DateTime @default(now()) @map("created_at")
  updatedAt       DateTime @updatedAt @map("updated_at")

  // 关系
  knowledgeBase   KnowledgeBase @relation(fields: [knowledgeBaseId], references: [id], onDelete: Cascade)
  chunks          Chunk[]

  @@unique([knowledgeBaseId, sourceId])
  @@index([knowledgeBaseId])
  @@map("knowledge_base_documents")

  createdAt DateTime @default(now()) @map("created_at")

  chunk     Chunk    @relation(fields: [chunkId], references: [id], onDelete: Cascade)

| Concern           | Mitigation                           |
| ----------------- | ------------------------------------ |
| 向量数据隔离      | 每个用户的向量存储在独立的 partition |
| 未授权访问        | 所有 API 验证用户身份和知识库所有权  |
| 数据泄露          | 向量数据不包含原文，仅用于相似度计算 |
| Google Drive 权限 | 仅访问用户明确选择的文件/文件夹      |

### 5.2 隐私保护

| Concern        | Mitigation                                      |
| -------------- | ----------------------------------------------- |
| 内容发送到 AI  | 仅发送相关 chunks，不发送完整文档               |
| 第三方数据共享 | 向量化使用 OpenAI API，遵循 OpenAI 数据使用政策 |
| 数据删除权     | 用户删除知识库时，彻底删除所有相关向量和 chunks |
