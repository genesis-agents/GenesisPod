# 产品规划整理

以下是对你这份规划要点的梳理，按主题归类并补充了内在逻辑，便于后续推进与对外沟通。

## 一、核心架构演进

**1. Token 压缩机制**
参考 RTK（Runtime Toolkit）方案，建立统一的 Token 压缩层，降低上下文成本并提升长链路推理稳定性。

**2. 分层架构重构（泛化下沉 / 业务上移）**
将通用能力（Harness、Engine、Infra、通用 Agent 编排）下沉到平台层，AI APP 层只保留与自身业务强相关的逻辑，做"薄应用"。目标是让 APP 易于复制、迭代和裁剪。

**3. CLI 全链路支持**
提供命令行入口，能够端到端驱动整个 Agent 体系（Harness → Engine → Infra），便于调试、自动化测试与 CI/CD 集成。

## 二、能力复用与生态

**4. Team 模式订阅**
将多 Agent 协作模式（Team）做成可订阅的能力包，新 APP 接入时直接选择对应 Team 模式即可完成集成，避免重复搭建。

**5. Workflow 订阅与可视化编排**
支持两种路径：一是订阅平台预置 Workflow；二是用户通过 UI 拖拽自行编排，编排结果可直接生成对应 Agent，并对接到第 2 点的薄应用层。也可考虑提供"对话式编排"作为更轻量的入口（可能比拖拽更自然）。

**7. Skill 导入与报告生成**
支持外部 Skill 导入，并能基于 Skill 生成多种形式的报告（文档、PPT、可视化等）。

## 三、运行与质量

**6. 本地模型支撑**
保证在本地模型上也能跑通完整链路，并产出**高质量报告**——这条是质量底线，需要专门的评测集来守护。

**8. 100% 自动化**
端到端全自动运行，无需人工介入中间环节。

## 四、闭环与多模态

**9. 内容发布与反馈闭环**
对接公众号、小红书等第三方平台，自动发布并回收阅读、互动等反馈数据，形成"生成 → 发布 → 反馈 → 优化"的系统闭环。

**10. 多模态输入**
支持语音输入；视频输入待评估（建议先明确场景与 ROI 再决定优先级）。
