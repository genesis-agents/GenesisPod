### BUG-001: 高亮数量与批注数量不匹配

**严重程度**: P0 严重
**复现步骤**:

1. 打开 Insights Report 页面
2. 查看批注面板显示的批注数量
3. 统计页面中 `<mark>` 元素数量

**预期结果**: 批注数量 = 高亮数量
**实际结果**:

- 批注面板显示: **17 个批注**
- 连续视图预览模式: **11 个高亮**
- 章节内容视图: **13 个高亮**

// 章节内容

**影响**: 用户创建的批注在原文中无法看到对应的高亮标记

## 技术分析

### 高亮元素样式

```css
mark {
  background-color: rgba(254, 240, 139, 0.7);
  display: inline;
  visibility: visible;
}
```

### 问题根因推测

1. **批注数据与DOM渲染不同步**: 批注存储的位置信息可能与实际DOM结构不匹配
2. **跨段落处理缺陷**: 当批注跨越多个DOM节点时，高亮渲染逻辑失败
3. **视图切换状态丢失**: 预览/编辑/章节视图切换时，批注渲染状态未正确传递
4. **Markdown渲染差异**: 不同视图模式下的Markdown渲染器可能产生不同的DOM结构
