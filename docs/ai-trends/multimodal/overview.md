# 多模态 AI

## 概述

多模态 AI 是指能够理解和生成多种类型数据（文本、图像、音频、视频等）的人工智能系统。随着 GPT-4V、Gemini 等模型的发布，多模态能力已成为大模型的标配。

### 2. 视觉编码器

```python
# 主流视觉编码器对比
vision_encoders = {
    "ViT (Vision Transformer)": {
        "原理": "图像分块 → Patch Embedding → Transformer",
        "优点": "与语言模型架构统一",
        "代表": "ViT-L/14, ViT-G/14",
    },

    "SigLIP": {
        "原理": "Sigmoid 对比学习 (替代 softmax)",
        "优点": "更高效的训练, 更好的 zero-shot",
        "代表": "PaLI, Gemini",
    },

    "CLIP ViT": {
        "原理": "对比学习预训练的视觉编码器",
        "优点": "强大的 zero-shot 能力",
        "代表": "OpenAI CLIP, OpenCLIP",
    },

    "ConvNeXt": {
        "原理": "现代化的 CNN 架构",
        "优点": "某些任务上更高效",
        "代表": "ConvNeXt-XXL",
    },
}

# 视觉 Token 生成
class VisionEncoder:
    def __init__(self, patch_size=14, hidden_dim=1024):
        self.patch_embed = PatchEmbed(patch_size, hidden_dim)
        self.transformer = ViT(hidden_dim, num_layers=24)

    def encode(self, image):
        # 图像 → Patches
        # (B, 3, 224, 224) → (B, 256, 1024)
        patches = self.patch_embed(image)

        # Transformer 编码
        visual_tokens = self.transformer(patches)

        return visual_tokens  # 输入到 LLM
```

### 3. 多模态融合策略

```
┌─────────────────────────────────────────────────────────────┐
│                  多模态融合方法                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. Early Fusion (早期融合)                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  [视觉 Token] + [文本 Token] → 统一 Transformer     │   │
│  │  优点: 深度交互                                      │   │
│  │  缺点: 计算量大                                      │   │
│  │  代表: Gemini, Claude 3                              │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  2. Late Fusion (晚期融合)                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  视觉分支 → 特征                                     │   │
│  │  文本分支 → 特征  → 融合层 → 输出                   │   │
│  │  优点: 模块化, 灵活                                  │   │
│  │  缺点: 交互不充分                                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  3. Cross Attention (交叉注意力)                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  文本 Token 通过 Cross-Attn 查询视觉特征            │   │
│  │  优点: 高效, 可控                                    │   │
│  │  代表: Flamingo, BLIP-2                             │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  4. Adapter/Projector                                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  视觉特征 → 线性投影 → LLM 嵌入空间                 │   │
│  │  优点: 简单高效, 保持 LLM 能力                       │   │
│  │  代表: LLaVA, Qwen-VL                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```
