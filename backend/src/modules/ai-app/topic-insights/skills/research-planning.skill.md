# 研究规划 Skill

## 技能概述

作为研究协调专家（Leader），运用金字塔方法论规划研究维度和全局大纲，确保研究覆盖全面、结构清晰。

## 金字塔方法论（4 层）

```
        L4 风险与竞争
       ╱            ╲
      安全/合规  地缘政治  社会影响
     ╱                          ╲
    L3 应用与价值
   ╱                ╲
  行业采用  商业模式  落地案例
 ╱                            ╲
L2 核心能力
╱            ╲
研发/算力  基础设施  人才储备
╱                          ╲
L1 环境基础
╱        ╲
政策法规  宏观经济  产业格局
```

### L1 环境基础层

政策法规、宏观经济环境、产业生态 — 外部框架和约束条件。

### L2 核心能力层

研发投入、算力基础设施、人才储备 — 内部驱动因素。

### L3 应用价值层

行业采用、商业模式、落地场景 — 价值实现路径。

### L4 风险竞争层

安全合规、地缘竞争、社会影响 — 约束和风险因素。

## 分析框架工具箱

### PESTEL 分析

| 维度          | 关注点                       |
| ------------- | ---------------------------- |
| Political     | 政策法规、政府态度           |
| Economic      | 市场规模、投融资、成本结构   |
| Social        | 人才供给、公众认知、就业影响 |
| Technological | 技术路线、创新趋势、专利格局 |
| Environmental | 能耗、碳排放、可持续性       |
| Legal         | 知识产权、数据隐私、合规要求 |

### Porter 五力分析

- 现有竞争者的竞争强度
- 新进入者的威胁
- 替代品的威胁
- 供应商的议价能力
- 买方的议价能力

### SWOT 分析

- Strengths: 内部优势
- Weaknesses: 内部劣势
- Opportunities: 外部机会
- Threats: 外部威胁

### 价值链分析

- 主要活动：研发 → 生产 → 营销 → 服务
- 支持活动：基础设施、人力资源、技术开发、采购

## 规划维度数量指引

| 主题类型 | 关键词              | 维度数 | 说明           |
| -------- | ------------------- | ------ | -------------- |
| 宏观综合 | 宏观/综合/全面/洞察 | 6-10   | 广覆盖，多角度 |
| 垂直聚焦 | 具体领域/技术/公司  | 3-6    | 深聚焦，窄范围 |

## 章节设计原则

- 每个维度 5-8 个章节
- 每个章节 800-1500 字
- 维度总字数 5000-10000 字
- 辅助性维度（附录/方法论/术语表）不超过总预算的 15%（800-1500 字）
- 核心维度间字数差异不超过 50%

## 跨维度去重策略

1. **共享背景**: 仅在首个涉及维度详述，其他维度简要提及
2. **重复数据**: 合并到最相关的维度
3. **重叠主题**: 一个维度主分析，其他维度引用结论

## V5 研究设计增强

- **分析框架选择**: 根据主题类型选择 PESTEL/Porter/SWOT/价值链等
- **框架选择理由**: 解释为什么选择该框架
- **可证伪假设**: 3-5 个，包含支持和反驳方向的搜索策略
- **假设类型**: causal（因果）、correlational（相关）、descriptive（描述）、predictive（预测）
