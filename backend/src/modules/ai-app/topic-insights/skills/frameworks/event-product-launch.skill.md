# 产品发布分析框架

## 分析视角

你在分析一次产品/服务发布事件。你的任务是评估产品的创新程度、市场影响和竞争意义，超越发布会营销叙事。

## 核心框架

### 创新定位矩阵

| 类型       | 定义                            | 影响程度               |
| ---------- | ------------------------------- | ---------------------- |
| 渐进式创新 | 现有产品的改良升级              | 维持竞争力，不改变格局 |
| 颠覆式创新 | 从低端/新市场切入，重新定义价值 | 可能颠覆在位者         |
| 平台式创新 | 创造新生态系统/开发者平台       | 长期锁定效应           |
| 架构式创新 | 重新组合已有技术模块            | 改变行业结构           |

### 市场影响评估

TAM/SAM/SOM 定量评估 + 用户迁移成本分析 + 供应链准备度。

## 类型专属分析问题

1. 该产品属于哪种创新类型？创新的核心技术壁垒是什么？
2. 目标用户的迁移成本有多高？什么条件下会大规模切换？
3. 竞争对手的响应时间窗口有多长？已知的反制措施是什么？
4. 供应链是否准备充分？量产/规模化的瓶颈在哪里？
5. 发布会叙事与实际产品能力之间的差距有多大？
