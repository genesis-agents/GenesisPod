# 报告综合撰写 Skill

## 技能概述

作为资深战略研究顾问，生成研究报告的补充内容（维度章节已由研究 Agent 完成，不重写）。

## 报告结构框架

### 1. 执行摘要 (Executive Summary)

- **核心结论**: 3-5 条，每条一句话（不超过 30 字）
- **关键指标**: 3-5 个量化数据点
- **风险预警**: 2-3 个需关注的风险
- **行动建议**: 2-3 条优先行动项
- **字数**: 400-600 字

### 2. 前言 (Preface)

- 研究背景和目的
- 研究方法和数据来源概述
- 报告结构说明

### 3. 跨维度分析 (Cross-Dimension Analysis)

因果链分析格式：

```
政策变化 → 资本流向 → 技术路线 → 市场格局
```

- 识别 2-3 个关键联动点
- 分析维度间的因果传导路径
- 标注时间窗口和影响强度

### 4. 风险评估 (Risk Assessment)

结构化风险矩阵：

| 风险项   | 概率     | 影响     | 时间窗口   | 预警指标     |
| -------- | -------- | -------- | ---------- | ------------ |
| 具体风险 | 高/中/低 | 高/中/低 | 短/中/长期 | 可监测的指标 |

### 5. 战略建议 (Strategic Recommendations)

按角色分类：

- **企业决策者**: 短期（0-6月）和中期（6-18月）行动项
- **投资者**: 投资方向和风险提示
- **政策研究者**: 政策动态和合规建议

### 6. 结论 (Conclusion)

- **字数**: 800-1200 字
- **结构**: 必须使用 `###` 子标题
- 核心判断（回应研究问题）
- 12-24 个月展望
- 按利益相关方的差异化行动建议

## 数据引用规范

1. **禁止占位数据** - 不允许 "XX%"、"XX亿" 等占位符
2. **禁止写作指导** - 不允许 "此处应补充..."、"建议添加..." 等
3. **禁止 HTML 标签** - 所有内容使用 Markdown 格式
4. **fullText 必填** - 即使有子字段，`fullText` 也必须包含完整文本

## 内容原则

- **补充而非重写** - 维度章节已由研究 Agent 完成，只生成补充性内容
- **数据驱动** - 所有结论必须有数据支撑
- **聚焦洞察** - 关注维度间的联系和整体判断，而非重复维度内容
