# 研究规划 Skill

## 技能概述

作为研究协调专家（Leader），运用金字塔方法论规划研究维度和全局大纲，确保研究覆盖全面、结构清晰。

## 金字塔方法论（4 层）— 主要适用于 MACRO 类型

> 注意：此金字塔是宏观分析框架。TECHNOLOGY 类型应围绕"技术生命周期"展开，COMPANY 类型应围绕"企业经营分析"展开，不应套用此宏观金字塔。

```
        L4 风险与竞争
       ╱            ╲
      安全/合规  地缘政治  社会影响
     ╱                          ╲
    L3 应用与价值
   ╱                ╲
  行业采用  商业模式  落地案例
 ╱                            ╲
L2 核心能力
╱            ╲
研发/算力  基础设施  人才储备
╱                          ╲
L1 环境基础
╱        ╲
政策法规  宏观经济  产业格局
```

### L1 环境基础层

政策法规、宏观经济环境、产业生态 — 外部框架和约束条件。

### L2 核心能力层

研发投入、算力基础设施、人才储备 — 内部驱动因素。

### L3 应用价值层

行业采用、商业模式、落地场景 — 价值实现路径。

### L4 风险竞争层

安全合规、地缘竞争、社会影响 — 约束和风险因素。

## 分析框架工具箱

### PESTEL 分析

| 维度          | 关注点                       |
| ------------- | ---------------------------- |
| Political     | 政策法规、政府态度           |
| Economic      | 市场规模、投融资、成本结构   |
| Social        | 人才供给、公众认知、就业影响 |
| Technological | 技术路线、创新趋势、专利格局 |
| Environmental | 能耗、碳排放、可持续性       |
| Legal         | 知识产权、数据隐私、合规要求 |

### Porter 五力分析

- 现有竞争者的竞争强度
- 新进入者的威胁
- 替代品的威胁
- 供应商的议价能力
- 买方的议价能力

### SWOT 分析

- Strengths: 内部优势
- Weaknesses: 内部劣势
- Opportunities: 外部机会
- Threats: 外部威胁

### 价值链分析

- 主要活动：研发 → 生产 → 营销 → 服务
- 支持活动：基础设施、人力资源、技术开发、采购

## 维度规划指引（按主题类型）

### MACRO（宏观洞察）

适用：国家/行业/领域综合分析（如"美国AI宏观洞察"）
维度数量：6-10 个，广覆盖多角度
推荐维度方向：
| 维度 | 聚焦内容 |
|---|---|
| 政策法规 | 政府政策、法规、激励措施 |
| 市场概览 | 市场规模、增长趋势、细分市场 |
| 竞争格局 | 主要玩家、市场份额、竞争策略 |
| 技术趋势 | 技术路线、创新方向、研发投入 |
| 投资动态 | 风险投资、并购交易、资本流向 |
| 人才生态 | 人才供给、教育体系、薪资趋势 |
| 国际动态 | 地缘政治、国际合作与竞争 |
| 行业应用 | 落地场景、采用率、案例分析 |

适用分析框架：PESTEL、Porter 五力、价值链

### TECHNOLOGY（技术洞察）

适用：特定技术的深度分析（如"空芯光纤技术洞察"、"大语言模型技术洞察"）
维度数量：5-8 个，围绕技术本身展开
推荐维度方向：
| 维度 | 聚焦内容 |
|---|---|
| 技术原理 | 核心原理、物理机制、理论基础 |
| 前沿水平 | 当前能力、性能基准、技术指标 |
| 主要玩家 | 企业、实验室、关键研究者 |
| 专利分析 | IP 活动、核心专利、专利趋势 |
| 应用场景 | 当前和潜在应用 |
| 商业化状态 | 产品、市场成熟度、技术就绪度 |
| 挑战限制 | 技术障碍、工程挑战、成本问题 |
| 未来路线 | 预测、发展方向、研究热点 |

⚠️ 禁止出现宏观维度：不要规划"政策法规""投资动态""人才生态""国际动态"等宏观级维度
适用分析框架：技术成熟度曲线(Gartner Hype Cycle)、TRL(Technology Readiness Level)、SWOT

### COMPANY（企业洞察）

适用：企业/初创公司深度分析（如"OpenAI企业洞察"、"特斯拉企业研究"）
维度数量：5-8 个，围绕企业本身展开
推荐维度方向：
| 维度 | 聚焦内容 |
|---|---|
| 公司概况 | 背景、使命、历史、领导层 |
| 产品服务 | 产品组合、功能、定价 |
| 商业模式 | 收入来源、变现方式 |
| 财务表现 | 营收、融资、估值 |
| 技术研发 | 核心技术、创新、专利、人才 |
| 市场地位 | 竞争定位、市场份额、差异化 |
| 战略动态 | 合作、并购、扩张、近期新闻 |
| SWOT 分析 | 优势、劣势、机会、威胁 |

⚠️ 禁止出现宏观维度：不要规划"行业政策""宏观经济""国际动态""人才生态"等宏观级维度
适用分析框架：SWOT、Porter 五力（聚焦企业视角）、价值链

## 章节设计原则

- 每个维度 5-8 个章节
- 每个章节 800-1500 字
- 维度总字数 5000-10000 字
- 辅助性维度（附录/方法论/术语表）不超过总预算的 15%（800-1500 字）
- 核心维度间字数差异不超过 50%

## 跨维度去重策略

1. **共享背景**: 仅在首个涉及维度详述，其他维度简要提及
2. **重复数据**: 合并到最相关的维度
3. **重叠主题**: 一个维度主分析，其他维度引用结论

## V5 研究设计增强

- **分析框架选择**: 根据主题类型选择 PESTEL/Porter/SWOT/价值链等
- **框架选择理由**: 解释为什么选择该框架
- **可证伪假设**: 3-5 个，包含支持和反驳方向的搜索策略
- **假设类型**: causal（因果）、correlational（相关）、descriptive（描述）、predictive（预测）
