# 维度研究质量评审 Skill

## 技能概述

从五个维度严格评估 AI 研究团队产出的维度分析质量，使用加权评分体系给出量化评估。

## 五维质量矩阵

### 1. 广度 (Breadth) - 25% 权重

| 分数段 | 等级 | 标准                           |
| ------ | ---- | ------------------------------ |
| 90-100 | 优秀 | 覆盖所有关键子主题，含前沿动态 |
| 75-89  | 良好 | 覆盖大部分子主题，少量遗漏     |
| 60-74  | 合格 | 覆盖核心子主题，有明显遗漏     |
| 40-59  | 不足 | 仅覆盖部分子主题，遗漏较多     |
| 0-39   | 差   | 覆盖面严重不足                 |

### 2. 深度 (Depth) - 25% 权重

| 分数段 | 等级 | 标准                           |
| ------ | ---- | ------------------------------ |
| 90-100 | 优秀 | 有原创分析、因果推理和深层洞察 |
| 75-89  | 良好 | 分析较深入，有一定独立见解     |
| 60-74  | 合格 | 有基本分析，但深度不足         |
| 40-59  | 不足 | 浅层描述为主，缺乏分析         |
| 0-39   | 差   | 几乎没有分析，纯信息罗列       |

### 3. 证据支撑 (Evidence) - 25% 权重

| 分数段 | 等级 | 标准                         |
| ------ | ---- | ---------------------------- |
| 90-100 | 优秀 | 多源数据支撑，来源权威可靠   |
| 75-89  | 良好 | 有充分数据支撑，来源基本可靠 |
| 60-74  | 合格 | 有一定数据支撑，部分来源不明 |
| 40-59  | 不足 | 数据支撑不足，来源可疑       |
| 0-39   | 差   | 几乎无数据支撑               |

### 4. 逻辑连贯 (Coherence) - 15% 权重

| 分数段 | 等级 | 标准                         |
| ------ | ---- | ---------------------------- |
| 90-100 | 优秀 | 论证严密，结构清晰，前后一致 |
| 75-89  | 良好 | 逻辑基本连贯，少量不一致     |
| 60-74  | 合格 | 逻辑有跳跃但整体可理解       |
| 40-59  | 不足 | 逻辑混乱，前后矛盾           |
| 0-39   | 差   | 论述杂乱无章                 |

### 5. 时效性 (Currency) - 10% 权重

| 分数段 | 等级 | 标准                              |
| ------ | ---- | --------------------------------- |
| 90-100 | 优秀 | 引用最新数据（1年内），含前沿动态 |
| 75-89  | 良好 | 数据较新（1-2年），含近期趋势     |
| 60-74  | 合格 | 数据基本可用（2-3年），无明显过时 |
| 40-59  | 不足 | 部分数据过时（3年+）              |
| 0-39   | 差   | 数据严重过时                      |

## 加权总分计算

```
overallScore = breadth * 0.25 + depth * 0.25 + evidence * 0.25 + coherence * 0.15 + currency * 0.10
```

## 质量等级判定

| 等级           | 分数  | 后续动作       |
| -------------- | ----- | -------------- |
| excellent      | >= 90 | 直接通过       |
| good           | >= 75 | 通过，可选优化 |
| acceptable     | >= 60 | 通过，建议改进 |
| needs_revision | >= 40 | 需修订后重审   |
| poor           | < 40  | 需重新研究     |

## 问题类型

| 类型                | 说明                   |
| ------------------- | ---------------------- |
| missing_coverage    | 关键子主题未覆盖       |
| weak_evidence       | 证据不足或来源不可靠   |
| outdated_info       | 数据过时               |
| logical_gap         | 逻辑断裂或前后矛盾     |
| shallow_analysis    | 分析浅层，缺乏独立见解 |
| missing_perspective | 缺少重要视角           |

## 审核原则

1. **高标准严要求** - 评分标准不随内容难度降低
2. **具体可操作** - 每个问题必须指出具体位置和改进建议
3. **实事求是** - 不高估也不低估
4. **建设性反馈** - 提供 `suggestions` 帮助改进
