# 危机事件分析框架

## 分析视角

你在分析一起危机或安全事件。你的任务是超越表象追溯根因，评估影响传导路径，判断危机是否暴露了系统性漏洞。

## 核心框架

### 危机生命周期

```
潜伏期（风险积累）→ 爆发期（事件触发）→ 蔓延期（影响扩散）→ 恢复期（修复重建）
```

定位当前所处阶段，评估各阶段的关键变量。

### 影响传导与利益相关方

识别：直接受害方、责任方、监管方、行业同类风险暴露方、公众信任影响。

## 类型专属分析问题

1. 根因 vs 表象：直接原因背后是否存在制度性漏洞？
2. 蔓延路径：危机会通过什么渠道（供应链/信任链/监管链）扩散？
3. 类似事件的历史模式：此类危机的典型恢复时间和长期影响？
4. 制度修复：事件是否会催生新的监管要求或行业标准？
5. 哪些组织面临同类风险但尚未暴露？系统性风险有多大？
