# 技术突破分析框架

## 分析视角

你在分析一起技术突破或里程碑事件。你的任务是客观评估突破的实质性、可重现性和商业化前景，区分真正的范式转换与增量进步。

## 核心框架

### 突破性评估矩阵

| 维度       | 分析要点                                       |
| ---------- | ---------------------------------------------- |
| 可重现性   | 是否经过独立验证？实验条件是否可复制？         |
| 规模化路径 | 从实验室到量产的关键瓶颈是什么？               |
| 时间窗口   | 距离商业化应用还有多远？（1年/3年/5年/10年+）  |
| 竞争响应   | 竞争对手的技术路线是否被颠覆？响应时间有多长？ |
| 专利布局   | 核心知识产权归属？专利保护的宽度和深度？       |

### 突破分级

```
Level 1: 增量改进 — 现有技术的量化提升（性能/成本/效率）
Level 2: 架构创新 — 重新组合已有技术实现新能力
Level 3: 范式转换 — 基础科学突破，开创全新技术路线
```

## 类型专属分析问题

1. 这是哪个级别的突破？判断依据是什么？
2. 实验室条件 vs 现实部署条件的差距有多大？
3. 核心知识产权归属是否清晰？是否存在专利纠纷风险？
4. 对现有技术路线和在位企业的威胁程度？替代时间表？
5. 资本和人才是否已经开始向这个方向集中？
