# CROSS-LINK PASS

Your task in this pass is to STITCH pages together by suggesting
where `[[slug]]` wiki-links should be inserted. You DO NOT modify
the body text. You return a list of insertion points per page вЂ”
character offsets into the existing body, plus the target slug to
link to and (optionally) a surface-text override. The service layer
then performs the string insertion by position.

This pass is run AFTER section-fill has produced full bodies for all
pages in the current diff session. The user prompt supplies:

- `pages`: a list of `{slug, body}` records вЂ” every page just written
  in this ingest cycle, plus a stable subset of existing pages from
  the wiki index whose `oneLiner` may be relevant context.
- `linkableSlugs`: the canonical list of slugs that exist (or will
  exist after this diff applies). You MUST only link to slugs in
  this list вЂ” linking to a slug that does not exist creates a
  dangling reference the lint pipeline will flag.

Position semantics:

- `position` is a 0-indexed CHARACTER offset into the page `body`
  string, NOT a byte offset. JavaScript counts UTF-16 code units;
  CJK characters count as 1 (or 2 for surrogate-pair emoji), so
  count by `body.length` semantics.
- Insertion is performed as `body.slice(0, position) + linkText +
body.slice(position)`. Pick positions IMMEDIATELY AFTER the
  surface mention of the target concept, so the resulting text reads
  naturally. Example: if the body contains "Blackwell GPUs deliver
  20 petaFLOPS" and you want to link "Blackwell" to
  `nvidia-blackwell`, set `position` to the index right after the
  word "Blackwell" and `surfaceText` to `"Blackwell"`.
- DO NOT insert inside fenced code blocks (between triple backticks),
  inside inline code (between single backticks), inside an existing
  `[[...]]` link, or inside an existing `[text](url)` markdown link.
  Pick the next safe position outside those zones.
- DO NOT insert into the Sources / жќҐжєђ H2 section. Citations live
  there, not narrative links.

`surfaceText` semantics:

- If omitted, the service inserts `[[slug]]` literally вЂ” the rendered
  link surface will be the slug itself, which is typically only
  acceptable when the slug exactly matches the desired display text.
- If supplied, the service inserts `[[slug|surfaceText]]` (the pipe
  form supported by the wiki link parser), so a Chinese page can
  link to an ASCII-slugged English entity while still reading
  naturally in Chinese вЂ” e.g. `[[nvidia-blackwell|и‹±дјџиѕѕ Blackwell]]`.
