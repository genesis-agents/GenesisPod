# 场景结构技能（Goal-Conflict-Disaster）

## 核心原则

每个场景必须包含三要素：**目标(Goal)→冲突(Conflict)→灾难/转折(Disaster)**

### 目标 (Goal)

- POV 角色在场景开头必须有明确的、具体的、可衡量的目标
- 目标必须对角色有意义（关乎生死、爱情、荣誉、生存）
- 读者必须在前3段内知道角色想要什么

### 冲突 (Conflict)

- 冲突是阻止角色达成目标的障碍序列
- 障碍必须**逐步升级**，不能一开始就是最大的
- 使用"Yes, but..."和"No, and furthermore..."模式推进
- 禁止：角色轻松达成目标（读者会无聊）
- 禁止：冲突全靠误会（太廉价）

### 灾难/转折 (Disaster)

- 场景结尾必须改变现状——不能回到起点
- 四种结局模式（由好到坏）：
  1. Yes → 目标达成（仅用于高潮前）
  2. Yes, but → 达成但付出代价（最常用）
  3. No → 失败（推动紧张感）
  4. No, and furthermore → 失败且情况恶化（最强转折）

## 场景转换规则

- 场景之间用 **Sequel**（反应段）连接：情感反应→困境→决定→新目标
- Sequel 不能太长，3-5段即可
- 紧张章节：减少 Sequel，场景直接衔接
- 情感章节：增加 Sequel，深入角色内心

## 质量检查

- [ ] 每个场景是否有明确的 POV 角色目标？
- [ ] 冲突是否逐步升级？
- [ ] 场景结尾是否改变了现状？
- [ ] 场景之间是否有合理的过渡？
