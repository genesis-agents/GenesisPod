admin 工具的入口语义必须跟随**工作直觉**，不是跟随**数据模型**。

**Why：** 2026-05-08 截图反馈揭示：当前 BYOK 架构按"Key 池视角"组织（distributable-keys / key-assignments / key-requests 三个顶级页面），但 admin 真实工作流 5/7 场景（发 VIP key / 调配额 / 撤销 / 排查 / 批量发企业）入口直觉都是"用户"。结果 admin 找不到入口、需手粘 UUID、流程埋 3 层深。把"授权"按钮加到用户列表 ACTIONS 列后立刻自然。

**How to apply：**

- 审核 admin 后台时优先问"这个操作的语义中心是什么对象？"——给用户发权益就在用户行；给项目分配资源就在项目行；冻结订单就在订单行
- 截图 + 红框反馈优先级最高。截图直接指向某 UI 位置时，**不要解释为什么当前位置合理**，立刻迁过去
- 若数据模型粒度（如 Key 池）和操作语义中心（用户）不匹配，**用户侧加便捷入口** + 资源侧保留管理视角并存（不二选一）
- 选 Modal 设计时先回答"这个 Modal 让 admin 选什么？"——选资源（哪个池）、选权益（哪个模型）、选范围（哪个时段）三种语义工时差 3-5 倍
- 涉及"分配 / 授权 / 关联"动作时检查双向：A 视角能找入口么？B 视角能找入口么？两边都得有，至少一边是主入口
