# 计划执行技能

> 系统性地执行计划中的任务，确保每个任务完成后得到验证。

## 执行原则

### 1. 一次一个任务

```
执行任务 N → 验证完成 → 报告状态 → 等待确认 → 执行任务 N+1
```

**禁止**：同时执行多个任务或跳过验证

### 2. 检查点机制

每完成一个任务后：

- [ ] 验证任务输出符合规格
- [ ] 运行相关测试
- [ ] 报告完成状态
- [ ] 等待确认后继续

### 3. 问题处理

遇到问题时：

1. **停止执行** - 不要继续
2. **记录问题** - 详细描述
3. **分析原因** - 根因分析
4. **提出方案** - 解决建议
5. **等待决策** - 人工确认

## 执行流程

### Phase 1: 加载计划

```bash
# 读取计划文件
Read: PLAN.md 或指定的计划文件

# 验证计划完整性
- 所有任务都有明确规格？
- 依赖关系清晰？
- 验证标准明确？
```

### Phase 2: 逐任务执行

对于每个任务：

```markdown
## 执行任务 N: [任务名称]

### 1. 前置检查

- [ ] 依赖任务已完成
- [ ] 所需输入可用
- [ ] 环境准备就绪

### 2. 执行步骤

[按计划中的步骤执行]

### 3. 输出验证

- [ ] 生成了预期输出
- [ ] 输出质量符合标准
- [ ] 相关测试通过

### 4. 状态报告

✅ 任务 N 完成

- 耗时: X 分钟
- 输出: [描述]
- 问题: [无/有描述]

等待确认后继续...
```

### Phase 3: 总结报告

所有任务完成后：

```markdown
## 执行总结

### 完成情况

- 总任务数: X
- 已完成: Y
- 跳过: Z
- 失败: W

### 问题记录

1. [问题描述和解决方案]

### 后续建议

1. [需要注意的事项]

### 验收清单

- [ ] 所有功能正常
- [ ] 测试全部通过
- [ ] 代码已审查
- [ ] 文档已更新
```

## 检查点类型

### 自动检查点

每个任务完成后自动触发：

- 类型检查
- 单元测试
- Lint 检查

### 人工检查点

关键节点需要人工确认：

- 架构变更
- API 接口变更
- 数据库迁移
- 安全相关变更

## 常见问题处理

### 任务失败

```markdown
## 任务 N 失败报告

### 失败原因

[详细描述]

### 错误信息

\`\`\`
[错误日志]
\`\`\`

### 分析

[根因分析]

### 建议方案

1. 方案 A: [描述]
2. 方案 B: [描述]

### 请选择

- [ ] 执行方案 A
- [ ] 执行方案 B
- [ ] 跳过此任务
- [ ] 中止执行
```

### 计划需调整

```markdown
## 计划调整建议

### 原因

[为什么需要调整]

### 建议变更

- 任务 X: [修改内容]
- 新增任务: [任务描述]

### 影响评估

- 时间影响: [评估]
- 依赖影响: [评估]

### 请确认

- [ ] 同意调整
- [ ] 保持原计划
```

## 执行命令

```bash
# 开始执行计划
/executing-plans PLAN.md

# 从特定任务开始
/executing-plans PLAN.md --from=3

# 只执行特定任务
/executing-plans PLAN.md --only=2,5,7
```
