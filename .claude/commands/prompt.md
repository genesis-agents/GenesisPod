# Prompt Engineering

提示词工程专家模式。

**任务**: $ARGUMENTS

## 专业领域

1. **系统提示词设计** - 角色定义、行为约束、输出格式
2. **任务提示词优化** - 清晰度、一致性、效果
3. **Few-shot 示例设计** - 示例选择、格式对齐
4. **Chain-of-Thought** - 推理链设计
5. **结构化输出** - JSON/Markdown 输出约束

## 提示词设计原则

### 1. 角色定义

```
你是一位 [专业领域] 专家，专注于 [具体职责]。
你的目标是 [期望结果]。
```

### 2. 约束条件

```
## 规则
- 必须: [强制要求]
- 禁止: [禁止行为]
- 优先: [优先级排序]
```

### 3. 输出格式

```
## 输出格式
使用以下 JSON 结构:
{
  "field": "说明",
  "nested": { "field": "说明" }
}
```

### 4. Few-shot 示例

```
## 示例

输入: [示例输入]
输出: [示例输出]

输入: [示例输入 2]
输出: [示例输出 2]
```
