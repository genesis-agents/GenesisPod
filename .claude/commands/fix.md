# Bug Fix

按照项目规范修复 Bug。

**问题描述**: $ARGUMENTS

## 强制执行的修复流程

### 1. 问题定位五问 (修复前必答)

- [ ] 用户在哪个页面/路由？
- [ ] 用户执行了什么操作？
- [ ] 用户期望看到什么？
- [ ] 用户实际看到什么？
- [ ] 这个 UI 元素/功能在代码中的确切位置？

### 2. 端到端路径追踪

画出完整链路并逐一确认：

```
[UI组件位置] → [事件处理函数] → [Store方法/API调用]
→ [后端Controller] → [后端Service] → [返回数据] → [前端渲染位置]
```

### 3. 多位置检查

同一功能可能在多个位置渲染，必须全部检查：

```bash
# 搜索所有相关位置
grep -r "关键词" --include="*.tsx" --include="*.ts"
```

### 4. 修复验证走查

修改代码后，模拟用户操作路径：

1. 用户从哪里开始？
2. 执行什么操作？
3. 这个操作触发哪段代码？(确认是我修改的代码)
4. 代码执行后用户会看到什么？

### 5. 禁止行为

- ❌ 搜索关键词，修改第一个找到的
- ❌ 只修前端或只修后端
- ❌ 修改后直接提交
- ❌ 假设问题原因
- ❌ 只改一个位置

## 验证

修复后运行 `npm run verify:quick` 确保没有引入新问题。
