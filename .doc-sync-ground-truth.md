## 1. 后端分层（唯一真值，来自 layer-boundaries.spec.ts）

```
L4 open-api   → 对外/外部访问面
L3 ai-app     → 业务应用
L2.5 ai-harness → 多 Agent 运行时
L2 ai-engine  → 通用 AI 基元
L1 platform   → 平台基础设施（概念层名 "ai-infra"，但真实目录是 platform/）
```

- **依赖方向（严格单向）**：`ai-app → ai-harness → ai-engine → platform`
  - ⚠️ 常见错误：写成 `ai-app → ai-engine → ai-harness → ai-infra`（engine/harness 顺序反了）
  - `open-api` 在最外层，可依赖任意下层；`common/` 是跨层共享工具；`platform` 不得反向依赖任何上层
- **L1 真实目录名是 `platform/`，不是 `ai-infra/`**。`backend/src/modules/ai-infra/` 这个路径**已不存在**。
  - 写法规范：描述层概念可写「L1 平台基础设施层（platform，旧称 ai-infra）」；写**文件路径**一律用 `backend/src/modules/platform/...`。

## 3. 插件系统（已落地，非空壳）
`backend/src/plugins/`：`core`（内核）、`observability`、`security`、`storage`（plugin 实现域）。
依赖规则见 standards/19：业务层只能 import `plugins/core`，不得 import 具体 `plugin:<domain>`（必须走 HookBus）。
- ⚠️ 文档若写「plugins 仍是 PR-0 空壳」= 陈旧，已有 4 个实现域。

## 4. 数据库（移除技术，重要）
- **唯一主库 PostgreSQL 16** + **Redis 7**（缓存/会话）。
- **已移除 MongoDB、Neo4j、Qdrant**（成本优化）。任何把 MongoDB/Neo4j/Qdrant 描述为「当前架构」的活跃文档都是陈旧。
  - ⚠️ `.claude/adrs/0003-dual-database-strategy.md` 仍把 PostgreSQL+MongoDB 双库写成当前决策 → 需标记 **Superseded**（ADR 不删除正文，加状态头 + 指向移除决策）。
- Prisma：目录模式 schema（`backend/prisma/schema/`）+ 手写 SQL 迁移（不用 `prisma migrate dev`）。

1. 只改 `.md`；不碰任何代码/入口文件。
2. 深度重写：对照本基准 + 实际目录，修正失真**叙述**，不只是替换字符串；逻辑陈旧的段落整段重写。
3. 不动 `_archive/` 与 `**/_archive/**`。
4. ADR 类文档不改写历史正文，只加状态标记（Superseded/Deprecated + 原因 + 日期）。
5. 区分前后端命名，不误伤前端合法名。
6. 不做范围外「顺手优化」。
